## 一、一句话定义

**Voice Keyboard 不是"又一个语音输入工具"。它是插在所有软件和操作系统之间的一层 AI 输入层——从此地球上每一个文本输入框都有了 AI 能力。**

按住按键说话，松手后文字出现在任意输入框。不只是语音转文字，更是**任何软件随时调用大模型**：微信聊天可以 AI 润色、邮件可以 AI 翻译、文档可以 AI 改写、代码可以 AI 生成——不需要任何软件做任何适配。

就像键盘的延伸。但这次，键盘有了大脑。

## 二、革命性之处——两个灵魂级差异化

> 这两个能力是整个项目最被低估的价值。不是"比竞品快 50ms"，而是"竞品根本没有这个维度"。


### 2.1 任何软件，都有 AI 了

这是 Voice Keyboard 最底层的范式创新。

微信没有 AI 助手。钉钉没有。邮件客户端没有。WordPress 后台编辑器没有。银行转账备注框没有。公司内部的 ERP、CRM、OA 系统——全都没有。

**但现在，它们全都有了。**

因为 Voice Keyboard 的 AI 不是"集成在某个应用里的功能插件"，而是**坐在操作系统和所有应用之间的那一层**。你在微信对话框里按住编辑键说"帮我把这段话改得更委婉一些"，LLM 自动润色后替换。你在邮件里说"把这段技术说明翻译成英文"，文字直接变成地道表达。你在任何地方，AI 都在。

**不需要任何软件做任何适配。不需要开放 API。不需要装插件。** 你打开了什么，Voice Keyboard 的 AI 就在哪里。

这就好比电力——你从不关心发电厂在哪，你只关心插上插座就有电。Voice Keyboard 就是所有文本输入框的 AI 插座。这个设计决策，让地球上每一个文本输入框一夜之间获得了大模型的能力。

| 以前 | 现在 |
|------|------|
| 微信聊天只能手动打字 | 微信聊天可以 AI 润色、翻译、改写 |
| 邮件客户端没有任何 AI | 写完邮件可以语音命令"改得更正式" |
| 文档编辑器需要装 AI 插件 | 直接说"把这段扩写到 500 字" |
| 代码注释要手打 | 说"给这个函数写中文注释" |
| 客服回复要复制粘贴到 ChatGPT | 直接在输入框里说"回得客气一点" |

**一句话：以前没法接入 AI 的软件，现在全都可以 AI 输入。**


### 2.2 越用越好用——它记住了你上一次的纠错

这是一个普通语音输入永远做不到的事：**记住你哪里容易出错，下次自动避免。**

你在微信里说"明天下午三点开会"。第一次，STT 识别成了"明天下午三天开会"。你按住编辑键说"三天改成三点"。Voice Keyboard 不仅修改了文字——它的核心模块 TextBuffer + LLM 记住了：你在说时间时，"三点"容易被识别为"三天"。

下一次，当你说"后天下午三点出发"，系统在 STT 识别后，LLM 做了常识校验："三天出发"在中文里语义不通。根据你的历史纠错模式，自动修正为"三点出发"，然后才打字。你什么都没做，但它已经替你修好了。

这就是"越用越好用"的底层逻辑——不是每次都从零开始，而是一个不断积累你个人语言模型的终身系统：

| 你每次使用 | 系统学到什么 |
|-----------|------------|
| 说"三点"被识别成"三天"→手动纠正 | 时间上下文中"三天"→"三点"自动修正 |
| 说"服务器"被识别成"服务期"→手动纠正 | 技术对话中"服务期"→"服务器"自动修正 |
| 说"李总"被识别成"你总"→手动纠正 | 称呼上下文中"你总"→"李总"自动修正 |
| 说英文人名被识别成中文→手动纠正 | 中英混排语境中保持英文原名不翻译 |

传统语音输入的尽头是"识别准确率 98%"。Voice Keyboard 的思考不一样：**剩下的 2% 怎么办？**

答案是——不靠声学模型死磕那 2%，而是用 LLM 理解语义，用 TextBuffer 追踪你的纠错历史，越用越准，越用越懂你。它不是一台打字机，它是一个越来越了解你的输入伙伴。

## 五、技术亮点（供对外宣传用）

### 4.1 "真正绕过输入法"的打字技术

大多数语音输入工具的问题是：文字会被输入法拦截，出现中英文乱码、繁简体转换等诡异问题。

Voice Keyboard 在三平台上用不同方案彻底绕过输入法：

- **macOS**：直接调用 Quartz `CGEventKeyboardSetUnicodeString` 向系统注入 Unicode 字符事件
- **Windows**：`SendInput + KEYEVENTF_UNICODE` 逐字发送，对微信/钉钉等 Electron 应用自动切换剪贴板粘贴模式
- **Linux**：X11 XTest 协议逐字输入

结果：无论什么应用（微信、钉钉、VS Code、浏览器、Word），文字都干净利落地出现，完全不经过 IME（输入法）。

### 4.2 五大 STT 引擎可选

| 引擎 | 延迟 | 中文能力 | 适合场景 |
|------|------|----------|---------|
| 讯飞语音听写 | 200-400ms | ★★★★ 逐字原文，数字自动转阿拉伯数字 | 中文日常推荐 |
| 阿里云 NLS | 200-400ms | ★★★★ 支持方言 | 方言用户 |
| 火山引擎 | 200-400ms | ★★★ 中文优化 | 字节生态用户 |
| 智谱 AI GLM-4-Voice | 500-800ms | ★★ 对话式改写 | 已有智谱 Key 的用户 |
| OpenAI Whisper | 500-1000ms | ★★ 多语言 | 英文/国际场景 |

推荐讯飞或阿里云——它们是专业 ASR 引擎，逐字原样转写，不乱加东西。

### 4.3 LLM 语音编辑 —— 这才是"生产力"

打完一句话后，按住编辑键说修改指令，LLM 自动执行。这是不是普通的"退格重打"：

| 你说的 | 效果 |
|--------|------|
| "把会议改成会谈" | 替换关键词 |
| "去掉最后一句" | 删除末尾内容 |
| "在后面加上请及时回复" | 追加文字 |
| "改成更正式的表达" | LLM 润色重写 |

技术上的精细之处：
- 有一个"文字账本"（TextBuffer）追踪自己打出的所有文字
- 同时监听键盘 Backspace 事件（30秒窗口），实时同步用户手动删除
- 监听鼠标点击事件——如果用户用过鼠标（光标可能位移了），编辑时会自动切换到"行选择模式"，通过 Home→Shift+End→复制 精确拿到当前行的真实内容，而不是依赖可能过时的文字账本

### 4.4 硬件架构 —— 无线 + 双模

硬件采用 **nRF52840（发射端）+ ESP32-S3（接收端）** 的双芯片方案：

- **发射端**：nRF52840，工作电流仅 12-18mA，500mAh 电池续航 **50+ 小时**。集成 PDM 数字 MEMS 麦克风，2 根线直连，近讲拾音。通过 BLE 5.0 传输 PCM 音频数据（16kHz/16bit/mono）。
- **接收端**：ESP32-S3，USB 复合设备——同时呈现 HID 键盘 + CDC 串口 + UAC 麦克风。按一个键即可在"语音打字模式"和"普通 USB 麦克风模式"之间切换。

为什么不用 ESP32 做发射端？ESP32 工作电流 80-150mA，同等电池只能用 3-4 小时，而 nRF52840 专为低功耗 BLE 设计。

设计哲学：设备本身**不连 WiFi，不做语音识别**。音频通过 USB → 电脑，由 Agent 调云端 API。设备换办公室/换 WiFi 零感知，STT 提供商和 API Key 在配置文件里随时切换，无需刷固件。

### 4.5 Claude Code 集成 —— 语音编程

按住 AI 热键（默认右 Shift）说："帮我把所有 print 改成 logging"，松手后 Claude Code 自动读取代码、批量替换，你的编辑器检测到文件变化自动刷新。

支持多轮对话——第一条指令建立上下文，后续指令自动续接，不需要每次重新说明背景。这是一个**真正的语音驱动 AI 编程助手**，不只是"语音转文字再手动粘贴给 ChatGPT"。

### 8.3 情感卖点

- **不打断思路**：灵感来的时候打字的节奏永远跟不上思考的速度。Voice Keyboard 让你可以专注于"想说什么"，而不是"怎么打出来"。
- **沉默的力量**：图书馆、深夜、公共场合——不用大声喊，近讲麦克风让你保持安静的体面。
- **键盘手的解放**：程序员每天敲击键盘数万次，Voice Keyboard 把注释、消息、文档的工作交给语音。

### 8.4 技术硬核点（适合科技媒体/KOL）

- 在 macOS 上用 Quartz Core Graphics 注入 Unicode 事件，彻底绕过 IME（输入法）
- Windows 上 GetForegroundWindow 判断当前窗口，自动在 SendInput 和剪贴板粘贴两种模式间切换

- BLE 5.0 自定义 GATT Service 传输 16kHz PCM 音频，有效带宽占用仅 18%
- ESP32-S3 USB 复合设备（HID + CDC + UAC），一个 USB 口同时是键盘、串口和麦克风

### 故事一 ·「凌晨两点的程序员」

**人物**：林晨，28 岁，全栈开发者，远程办公

凌晨两点，林晨盯着屏幕上那个 bug。他已经沉默地翻了一个小时的代码，手腕开始隐隐发酸——上个月体检报告说"早期腕管综合征"的时候，他没太当回事。

他突然想到问题出在哪了。

他的手没有伸向键盘。他对着衣领上那枚指甲盖大的麦克风，像自言自语一样说了一句话："在 handlePayment 函数里，把回调里的硬编码超时时间从 5000 改成从 config 读取，加上 try-catch，失败时打 warn 日志而不是 throw。"

三秒后，终端里 Claude Code 开始逐行输出修改内容。文件自动保存，编辑器刷新，新代码已经在屏幕上了。

全程没碰一次键盘。没碰一次鼠标。

他又说了一句："给这个改动写一个单元测试，覆盖成功和超时两种情况。"

半分钟后，`handlePayment.test.ts` 出现在侧边栏。他端起已经凉了的咖啡——这是他今晚第一次感觉到，手腕不疼了。

不只是今晚。是从装上 Voice Keyboard 之后，他就再也没在深夜揉过手腕。

### 故事二 ·「凌晨四点的新手妈妈」

**人物**：苏然，31 岁，内容运营总监，宝宝 3 个月大

凌晨 3:47，喂完奶、拍完嗝、重新哄睡，指针指向 4:12。

27 条未读微信。明天上午产品发布会，全是最终确认事项。往常她会在黑暗中摸到手机，噼里啪啦打字——键盘声吵醒宝宝，新一轮哭声，然后彻底告别今晚最后一点睡眠。

今晚不一样。衣领上别着 Voice Keyboard 发射端。手机屏幕亮着，微信对话框开着。

她用气声，几乎只有嘴唇在动："文案终稿已确认，通知设计部明天八点前出最后一批 banner。媒体通稿第三段有笔误，把'预计增长 30%'改成'预计增长 30% 以上'。会后帮我约李总单独沟通预算。"

文字悄无声息地出现在微信输入框里。她轻声说了一句"发送"——系统识别为快捷键指令，消息发出。

宝宝翻了个身，继续睡。苏然也翻了个身，离天亮还有两个半小时。

从始至终，键盘没有响过一声。

### 故事三 ·「六十七岁的第一天」

**人物**：陈爷爷，67 岁，退休工程师

孙子教他用微信三年了。他喜欢看孙子发来的文字消息，也想用文字回复，但每次打一行字要五分钟——拼音不太准，手写识别慢。

语音消息他试过。但总觉得"太吵了，不像文字那么正式"。

孙子送了他一个盒子。"爷爷你把这个小东西插到电脑上，然后直接说话就行。不用按任何键。"

他把那个指甲盖大的小方块夹在衣领上，试着说了一句："明明，周末回来吃饭吗？奶奶做了你爱吃的红烧肉。"

文字出现在微信输入框里。一个字不错。"明明"没写成"明明"，"红烧肉"没写成"红绕肉"。他甚至没说"发送"——文字输完之后，他习惯性地用鼠标点了一下发送键——但其实说一句"回车"也能发。

他盯着屏幕愣了三秒，笑得像个孩子。"这个好，这个好。"

那天下午，陈爷爷给七个老同事发了文字消息，还给儿子发了一段两百字的"最近身体怎么样，工作别太累"。全部用说的。

他后来跟小区的老伙计们炫耀："我现在打字比你们谁都快。因为我根本不打。"

### 故事四 ·「同时活在两个会议室里的人」

**人物**：Katherine，35 岁，跨国团队产品负责人

下午三点，Zoom 视频会议。Katherine 正在讲下季度的需求优先级。她的摄像头开着，德国同事看着她的脸。

同时，微信上合伙人连发了五条消息问她 Q3 预算数字。每一条都标注"急"。

往常她只有三个选择：低头打字（对方看到你眼神飘忽）、关掉视频打字再切回来（打断自己正在讲的逻辑）、或者假装没看到（让合伙人干等半小时）。

今天她选第四种。她保持着眼神看向摄像头，语调平稳地讲着需求，手自然地放在桌面上。嘴唇几乎不动，用只有自己能听到的音量说："Q3 研发预算 380 万，市场预算 120 万，具体拆解见共享文档第三页。"

两秒后，微信对话框里出现了这段话。她说了一声"发送"，消息发出。

前后不超过四秒。屏幕对面的德国同事不知道，屏幕这边的合伙人也不知道——她刚才同时活在两个会议室里。

会后的录音转文字里，她的发言连贯、专业、毫无破绽。股东群里的预算数字，精准到万。

### 故事五 ·「语速终于追上思考的人」

**人物**：阿 Yan，26 岁，自由撰稿人 & B 站 UP 主

阿 Yan 有一个执念：最好的文字不是"写"出来的，是"流"出来的。

残酷的现实：语速 180 字/分钟，打字速度 50 字/分钟。灵感来的时候，他的手永远追不上脑子。打完前半句，后半句的措辞已经忘了。

他试过语音备忘录转文字。流程是：录完→导出→识别→复制→粘贴→排版。中间每一步都在"打断"。灵感断了就接不上。

今天他在星巴克角落坐下，打开 VS Code——纯文本、无干扰。Voice Keyboard 发射端夹在鸭舌帽檐内侧，麦克风刚好对着嘴角。接收端插在笔记本 USB 口上。

他说话。电脑打字。

"中国独立创作者正在经历一场静默的身份转变。十年前我们叫自己'接外包的'，五年前叫'自由职业者'，现在越来越多的人开始认真使用一个词——"

文字出现在编辑器里，标点正确，中英文混排无乱码。他看了一眼，又说："把'身份转变'改成'叙事升级'。"

两个字被精准替换。

他接着写。两个小时后站起来买第二杯咖啡，看了一眼屏幕左下角：3476 字。

他在当天的 vlog 里说："今天写稿的速度，终于追上我想事情的速度了。不是比喻，是字面意思。"

### 故事六 ·「没有人敲键盘的办公室」

**人物**：三个人的 SaaS 创业团队，挤在共享办公室的角落里

Jason（后端）盯着一个并发 bug，突然开口："给 userSync 方法加上分布式锁，用 Redis 实现，锁超时 30 秒，失败重试三次。"

Claude Code 开始改代码。Jason 的手没离开过咖啡杯。

Mia（运营）同时开着五个微信对话框。她不看键盘，对着屏幕说："今天下午会推送版本更新，您反馈的三个问题已全部修复。切换到下一个对话。感谢您的耐心等待，我们会持续优化体验。发送。"

每到一个对话框，她就说一句，文字自动出现在输入框里。说"回车"就发送，说"下一个"就用快捷键切窗口。

老周（创始人）在写融资 BP。他盯着天花板想了 20 秒，然后开口："请用更简洁的商业语言重写这一段，保留所有数据，去掉形容词。"

三个人在同一张桌子上，各自对着屏幕说话。没有人敲键盘。只有偶尔的笑声和键盘声——那是 Mia 忍不住手动回复了一个特别离谱的客户。

一个路过的大叔好奇地探头看了一眼："你们这办公室怎么这么安静？没人敲键盘？"

老周抬头笑了笑，说了一句话。办公室墙上挂的大屏幕出现了三个字：

"因为我们说话就够了。"

### 关于硬件模式的叙事说明

以上所有故事描述的是**硬件模式下的完整体验**——发射端佩戴在身上，接收端插在电脑 USB 口，VAD 自动检测语音边界，全程无需按键。叙事中刻意避免了三件事：

1. **不提"按住某键"**——硬件 VAD 的自动唤醒是全产品线最核心的体验差异。近讲麦克风贴在嘴边 3-5cm，信噪比极高，只有你的声音能触发，环境噪音不会。
2. **展示"语音操控电脑"**——故事里出现了"发送"切窗口"回车上一个"等语音快捷键。Voice Keyboard 可以发出完整的系统快捷键序列（Ctrl+Tab 切标签、Win+数字 切应用、Alt+Tab 切窗口等），让你用声音在应用之间跳转，比鼠标还快。
3. **键盘只保留"必要时刻"**——编程时用 AI 模式改代码（不用键盘）、写稿时语音输出、回消息时语音输入……只有在细微调整或下意识动作时才摸键盘。键盘从"主要输入设备"退居为"偶尔的辅助工具"。

> 市场部创作内容时请注意：叙事中不要出现"按住 Alt""松开按键""按下热键"等操作描述。产品的最强叙事是"你在说话，电脑在打字，中间没有按键、没有鼠标、没有等待。就是说话。"
